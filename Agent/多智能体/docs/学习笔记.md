# 多智能体工程级学习笔记（中文 + LangGraph + A2A + 千问）

## 1. 你现在拿到的是什么
这是一个“新手友好、可直接运行、可逐步扩展”的多智能体学习工程。

核心目标：
- 用 LangGraph 学会“状态图编排”。
- 用 A2A 思路学会“跨 Agent 协议化协作”。
- 支持阿里云千问（DashScope OpenAI 兼容接口）。
- 即使你还没配置 API Key，项目也能离线运行（自动回退）。

## 2. 目录导读
- `src/multi_agent/llm_qwen.py`：千问模型调用封装（可远程、可回退）。
- `src/multi_agent/agents.py`：数学/研究/评审 Agent + Supervisor 路由。
- `src/multi_agent/langgraph_workflow.py`：LangGraph 状态图定义与执行入口。
- `src/multi_agent/a2a/protocol.py`：A2A 协议对象（Task、Message、Artifact、AgentCard）。
- `src/multi_agent/a2a/orchestrator.py`：A2A 四种典型模式（顺序、并行、条件、流水线）。
- `scripts/run_demo.py`：中文演示脚本。
- `scripts/direct_check.py`：直接执行检查脚本（不依赖 unittest）。

## 3. 如何接入阿里云千问

### 3.1 先配置环境变量
PowerShell 示例：

```powershell
$env:DASHSCOPE_API_KEY="你的阿里云DashScope密钥"
$env:QWEN_MODEL="qwen-plus"
# 可选：默认已经是这个地址，不配也能用
$env:DASHSCOPE_BASE_URL="https://dashscope.aliyuncs.com/compatible-mode/v1"
```

### 3.2 运行演示

```powershell
D:\dev\Python\Agent\.venv\Scripts\python.exe D:\dev\Python\Agent\多智能体\scripts\run_demo.py
```

如果没有配置 `DASHSCOPE_API_KEY`，代码会自动走本地回退文本，保证你仍然能学习流程。

## 4. A2A 四种模式你要重点理解什么
1. 顺序：前一步结果给后一步，适合线性审批流。
2. 并行：多个 Agent 同时做，最后汇总，适合提速。
3. 条件：根据输入动态选路，适合意图路由和降本。
4. 流水线：固定阶段串联，每阶段产出可追踪 Artifact。

## 5. 为什么这套实现适合新手
- 代码里是高密度中文注释。
- 运行入口非常直接：`run_demo.py` / `direct_check.py`。
- 先本地可跑，再无缝切换到真实千问模型。
- LangGraph 和 A2A 分层清晰，后续改造不容易乱。
