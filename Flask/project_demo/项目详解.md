## 1. 项目结构

```
project_demo/
├── config.py              # 配置对象 - 管理不同环境的配置
├── run.py                 # 入口文件 - 启动应用
├── requirements.txt       # 依赖包列表
└── app/                   # 应用主目录
    ├── __init__.py        # 应用工厂 - 创建和配置应用
    ├── models.py          # 数据库模型 - 定义表结构
    ├── main.py            # 主蓝图 - 首页等通用路由
    └── user.py            # 用户蓝图 - 用户相关的 API
```

**为什么这样组织？**
- 📦 **模块化**：每个功能独立，便于维护
- 🔄 **可复用**：蓝图可以在不同项目中复用
- 🧪 **易测试**：应用工厂可以创建多个独立实例
- 🌍 **多环境**：配置对象支持开发/测试/生产环境

## 2. 核心概念详解

### 2.1 什么是应用工厂？

#### 💡 概念

**应用工厂**就是一个**创建 Flask 应用的函数**。就像工厂生产产品一样，这个函数"生产"Flask 应用实例。

#### ❓ 为什么要用应用工厂？

**传统方式（不用工厂）：**
```python
# app.py
from flask import Flask

app = Flask(__name__)  # 直接创建，全局变量

@app.route('/')
def index():
    return 'Hello'

if __name__ == '__main__':
    app.run()
```

**问题：**
- ❌ 只有一个全局 `app` 对象，难以创建多个实例
- ❌ 测试时难以隔离
- ❌ 扩展（如数据库）初始化混乱
- ❌ 无法动态配置

**应用工厂方式：**
```python
# 工厂函数
def create_app(config_name):
    app = Flask(__name__)
    # 配置、注册蓝图、初始化扩展...
    return app

# 可以随时创建新实例
app1 = create_app('development')
app2 = create_app('testing')
```

**优点：**
- ✅ 可以创建多个独立的应用实例
- ✅ 测试时每个测试用例可以用独立实例
- ✅ 扩展初始化更清晰
- ✅ 支持动态配置（开发/生产环境）

#### 📝 应用工厂 Demo

```python
from flask import Flask

def create_app(config_name='default'):
    """应用工厂函数"""
    # 1. 创建 Flask 实例
    app = Flask(__name__)
    
    # 2. 加载配置
    if config_name == 'development':
        app.config['DEBUG'] = True
    elif config_name == 'production':
        app.config['DEBUG'] = False
    
    # 3. 注册蓝图（路由模块）
    from routes import main_bp
    app.register_blueprint(main_bp)
    
    # 4. 初始化扩展（数据库等）
    # db.init_app(app)
    
    # 5. 返回配置好的应用
    return app

# 使用工厂创建应用
app = create_app('development')

if __name__ == '__main__':
    app.run()
```

### 2.2 什么是配置对象？

#### 💡 概念

**配置对象**就是用 Python 类来管理配置信息，而不是把配置写死在代码里或放在配置文件中。

#### ❓ 为什么要用配置对象？

**问题场景：**
- 开发时：需要 DEBUG=True，用本地数据库
- 测试时：需要 TESTING=True，用测试数据库
- 生产时：需要 DEBUG=False，用生产数据库

**不用配置对象：**
```python
# 到处是 if-else，难以维护
if environment == 'dev':
    app.config['DEBUG'] = True
    app.config['DATABASE'] = 'sqlite:///dev.db'
elif environment == 'prod':
    app.config['DEBUG'] = False
    app.config['DATABASE'] = 'mysql://prod-server/db'
```

**使用配置对象：**
```python
# 清晰、面向对象、易维护
class DevelopmentConfig:
    DEBUG = True
    DATABASE = 'sqlite:///dev.db'

class ProductionConfig:
    DEBUG = False
    DATABASE = 'mysql://prod-server/db'
```

#### 📝 配置对象 Demo

```python
import os

class Config:
    """基础配置（所有环境共享）"""
    SECRET_KEY = 'hard-to-guess'  # 密钥
    SQLALCHEMY_TRACK_MODIFICATIONS = False

class DevelopmentConfig(Config):
    """开发环境配置"""
    DEBUG = True  # 开启调试模式
    DATABASE_URL = 'sqlite:///dev.db'  # 开发用数据库

class TestingConfig(Config):
    """测试环境配置"""
    TESTING = True  # 开启测试模式
    DATABASE_URL = 'sqlite:///test.db'  # 测试用数据库
    WTF_CSRF_ENABLED = False  # 关闭 CSRF 保护

class ProductionConfig(Config):
    """生产环境配置"""
    DEBUG = False  # 关闭调试
    DATABASE_URL = os.environ.get('DATABASE_URL')  # 从环境变量读取

# 配置字典 - 方便根据名称获取配置类
config = {
    'development': DevelopmentConfig,
    'testing': TestingConfig,
    'production': ProductionConfig,
    'default': DevelopmentConfig
}

# 在应用工厂中使用
def create_app(config_name='default'):
    app = Flask(__name__)
    
    # 一行代码加载配置
    app.config.from_object(config[config_name])
    
    return app
```

**关键方法：**
- `app.config.from_object()`: 从配置对象加载所有大写字母的变量
- `os.environ.get()`: 从环境变量读取（生产环境推荐）

### 2.3 什么是蓝图？

#### 💡 概念

**蓝图（Blueprint）**是 Flask 的**路由模块化**工具。它允许你把路由分组到不同的模块中，最后再注册到主应用上。

#### ❓ 为什么要用蓝图？

**问题场景：**
假设你的应用有 100 个路由，全部写在一个文件里：
```python
# app.py - 1000+ 行，难以维护
@app.route('/')
def index(): ...

@app.route('/user/list')
def user_list(): ...

@app.route('/user/create')
def user_create(): ...

@app.route('/product/list')
def product_list(): ...

@app.route('/product/create')
def product_create(): ...

# ... 还有 95 个路由 ...
```

**使用蓝图：**
```
app/
├── main.py      # 首页蓝图（5 个路由）
├── user.py      # 用户蓝图（20 个路由）
└── product.py   # 商品蓝图（30 个路由）
```

#### 📝 蓝图 Demo

**1. 创建蓝图（user.py）：**
```python
from flask import Blueprint

# 创建蓝图
# 参数：蓝图名称, 导入名称, url_prefix=URL 前缀
user_bp = Blueprint('user', __name__, url_prefix='/api/user')

# 在蓝图上定义路由（注意：用 @user_bp.route 而不是 @app.route）
@user_bp.route('/list')
def user_list():
    return '用户列表'

@user_bp.route('/<int:user_id>')
def user_detail(user_id):
    return f'用户详情: {user_id}'
```

**2. 注册蓝图到应用（__init__.py）：**
```python
from flask import Flask

def create_app():
    app = Flask(__name__)
    
    # 导入蓝图
    from app.user import user_bp
    
    # 注册蓝图
    app.register_blueprint(user_bp)
    
    return app
```

**3. 访问路由：**
```
GET /api/user/list         -> 用户列表
GET /api/user/1            -> 用户详情（user_id=1）
```

**注意：** 因为有 `url_prefix='/api/user'`，所以实际路径会自动加上前缀。

#### 🎯 蓝图的核心优势

1. **模块化**：每个功能独立文件
2. **URL 前缀**：自动添加路径前缀
3. **可复用**：蓝图可以在多个项目中复用
4. **团队协作**：不同开发者负责不同蓝图
5. **易于测试**：可以单独测试某个蓝图

### 3.2 app/__init__.py - 应用工厂

```python
from flask import Flask
from flask_sqlalchemy import SQLAlchemy
from config import config  # 导入配置字典

# 创建 SQLAlchemy 实例（但不绑定到具体应用）
db = SQLAlchemy()


def create_app(config_name='default'):
    """
    应用工厂函数
    
    参数：
        config_name: 配置名称，可选 'development', 'testing', 'production'
    
    返回：
        配置好的 Flask 应用实例
    """
    # 1. 创建 Flask 应用实例
    # __name__ 表示当前模块名，Flask 用它确定资源文件的位置
    app = Flask(__name__)
    
    # 2. 从配置对象加载配置
    # 例如 config_name='development' 会加载 DevelopmentConfig 的所有大写变量
    app.config.from_object(config[config_name])
    
    # 3. 初始化数据库扩展
    # 将 db 与当前 app 实例绑定
    db.init_app(app)
    
    # 4. 注册蓝图 - 主蓝图
    # 延迟导入：避免循环依赖
    from app.main import main as main_blueprint
    app.register_blueprint(main_blueprint)
    
    # 5. 注册蓝图 - 用户蓝图
    from app.user import user as user_blueprint
    app.register_blueprint(user_blueprint)
    
    # 6. 创建数据库表
    # app.app_context()：创建应用上下文，数据库操作需要这个上下文
    with app.app_context():
        from app.models import User  # 导入模型
        db.create_all()  # 如果表不存在则创建
    
    # 7. 返回配置好的应用
    return app
```

**关键概念：**
- **延迟导入**：在函数内部导入，避免循环依赖
- **应用上下文**：Flask 的某些操作（如数据库）需要在应用上下文中执行
- **db.init_app()**：将扩展与应用绑定，这是应用工厂模式的标准做法

### 3.3 app/models.py - 数据库模型

```python
from flask_sqlalchemy import SQLAlchemy
from datetime import datetime

# 从 app/__init__.py 导入已创建的 db 实例
from app import db


class User(db.Model):
    """
    用户模型 - 对应数据库中的 user 表
    
    db.Model 是 SQLAlchemy 提供的基类
    """
    # 指定表名（可选，如果不指定会用类名的小写形式）
    __tablename__ = 'user'
    
    # 定义字段 - 对应数据库的列
    # db.Column: 定义列
    # 第一个参数是数据类型，后面是约束条件
    id = db.Column(db.Integer, primary_key=True, autoincrement=True)
    username = db.Column(db.String(20), nullable=False, unique=True)
    password = db.Column(db.String(20), nullable=False)
    email = db.Column(db.String(20), nullable=False, unique=True)
    phone = db.Column(db.String(20), nullable=False, unique=True)
    sex = db.Column(db.String(20), nullable=False)
    birthday = db.Column(db.String(20), nullable=False)
    avatar = db.Column(db.String(20), nullable=False)
    create_time = db.Column(db.DateTime, nullable=False, default=datetime.now)
    
    def to_dict(self):
        """
        将模型实例转换为字典
        
        用途：API 返回 JSON 数据时需要将对象序列化
        """
        return {
            'id': self.id,
            'username': self.username,
            'email': self.email,
            'phone': self.phone,
            'sex': self.sex,
            'birthday': self.birthday,
            'avatar': self.avatar,
            'create_time': self.create_time.strftime('%Y-%m-%d %H:%M:%S') if self.create_time else None
        }
    
    def __repr__(self):
        """
        对象的字符串表示
        
        用途：调试时打印对象会显示这个字符串
        例如：print(user) 输出 <User 张三>
        """
        return f'<User {self.username}>'
```

**字段约束说明：**
- `primary_key=True`: 主键
- `autoincrement=True`: 自动递增
- `nullable=False`: 不允许为空
- `unique=True`: 值必须唯一
- `default=datetime.now`: 默认值为当前时间

### 3.5 app/user.py - 用户蓝图（核心）

```python
from flask import Blueprint, request, jsonify
from app import db
from app.models import User

# 创建用户蓝图
# url_prefix='/api/user': 所有路由都会加上这个前缀
user = Blueprint('user', __name__, url_prefix='/api/user')


@user.route('/list', methods=['GET'])
def user_list():
    """
    获取所有用户列表
    
    methods=['GET']: 只允许 GET 请求
    """
    # User.query.all(): 查询所有用户（SQLAlchemy 的查询方法）
    users = User.query.all()
    
    # 返回 JSON 响应
    # jsonify: 将 Python 字典转换为 JSON 响应
    return jsonify({
        'code': 200,              # 状态码
        'message': 'success',     # 消息
        'data': [user.to_dict() for user in users]  # 数据（列表推导式）
    })


@user.route('/<int:user_id>', methods=['GET'])
def get_user(user_id):
    """
    获取单个用户
    
    <int:user_id>: 路径参数，会自动转换为整数
    """
    # get_or_404: 如果找不到返回 404 错误
    user = User.query.get_or_404(user_id)
    
    return jsonify({
        'code': 200,
        'message': 'success',
        'data': user.to_dict()
    })


@user.route('/create', methods=['POST'])
def create_user():
    """
    创建用户
    
    methods=['POST']: 允许 POST 请求
    """
    # request.get_json(): 获取请求体中的 JSON 数据
    data = request.get_json()
    
    # 验证必填字段
    required_fields = ['username', 'password', 'email', 'phone', 'sex', 'birthday', 'avatar']
    for field in required_fields:
        if field not in data:
            return jsonify({
                'code': 400,
                'message': f'缺少必填字段: {field}'
            }), 400  # 第二个返回值是 HTTP 状态码
    
    # 检查用户名是否已存在
    # filter_by: 条件过滤
    # first(): 返回第一条记录，如果没有返回 None
    if User.query.filter_by(username=data['username']).first():
        return jsonify({
            'code': 400,
            'message': '用户名已存在'
        }), 400
    
    # 创建新用户实例
    new_user = User(
        username=data['username'],
        password=data['password'],
        email=data['email'],
        phone=data['phone'],
        sex=data['sex'],
        birthday=data['birthday'],
        avatar=data['avatar']
    )
    
    # 添加到数据库会话
    db.session.add(new_user)
    # 提交事务（真正写入数据库）
    db.session.commit()
    
    return jsonify({
        'code': 200,
        'message': '创建成功',
        'data': new_user.to_dict()
    }), 201  # 201 表示资源创建成功


@user.route('/<int:user_id>', methods=['PUT'])
def update_user(user_id):
    """
    更新用户
    
    methods=['PUT']: 允许 PUT 请求（用于更新）
    """
    # 查找用户
    user = User.query.get_or_404(user_id)
    data = request.get_json()
    
    # 只更新提供的字段
    if 'username' in data:
        user.username = data['username']
    if 'password' in data:
        user.password = data['password']
    if 'email' in data:
        user.email = data['email']
    # ... 其他字段同理
    
    # 提交更改
    db.session.commit()
    
    return jsonify({
        'code': 200,
        'message': '更新成功',
        'data': user.to_dict()
    })


@user.route('/<int:user_id>', methods=['DELETE'])
def delete_user(user_id):
    """
    删除用户
    
    methods=['DELETE']: 允许 DELETE 请求
    """
    user = User.query.get_or_404(user_id)
    
    # 从会话中删除
    db.session.delete(user)
    # 提交事务
    db.session.commit()
    
    return jsonify({
        'code': 200,
        'message': '删除成功'
    })
```

**RESTful API 设计原则：**
- `GET` - 获取资源
- `POST` - 创建资源
- `PUT` - 更新资源
- `DELETE` - 删除资源

## 6. 常见问题

### Q1: 为什么要用 `db.init_app(app)` 而不是直接 `db = SQLAlchemy(app)`？

**A:** 这是应用工厂模式的要求。因为 `db` 对象在创建应用之前就需要定义，但此时还没有 `app` 实例，所以先创建 `db`，然后在工厂函数中用 `init_app()` 绑定。

```python
# ❌ 错误写法（不能用工厂模式）
db = SQLAlchemy(app)

# ✅ 正确写法（应用工厂模式）
db = SQLAlchemy()

def create_app():
    app = Flask(__name__)
    db.init_app(app)  # 延迟绑定
    return app
```

---

### Q2: 什么是应用上下文（app_context）？

**A:** Flask 的某些操作（如数据库查询）需要在应用上下文中执行。简单来说，就是告诉 Flask "我现在在使用哪个应用"。

```python
# 在应用工厂中创建表时需要
with app.app_context():
    db.create_all()  # 这个操作需要应用上下文

# 在路由函数中不需要手动创建
# 因为 Flask 会自动处理
@user.route('/list')
def user_list():
    users = User.query.all()  # 自动在上下文中
    return jsonify(...)
```

---

### Q3: 为什么要在函数内部导入蓝图？

**A:** 避免**循环导入**问题。

```python
# ❌ 错误写法（会导致循环导入）
# __init__.py
from app.main import main  # 导入 main
main.py
from app import db  # 导入 db（但 db 在 __init__.py 中）
# 互相导入，死循环！

# ✅ 正确写法（延迟导入）
def create_app():
    # 在函数内部导入，此时 db 已经定义
    from app.main import main
    app.register_blueprint(main)
```

---

### Q4: 如何切换到生产环境？

**A:** 修改 `run.py` 中的配置名称：

```python
# 开发环境
app = create_app('development')

# 生产环境
app = create_app('production')
```

生产环境建议：
- 关闭 DEBUG
- 使用环境变量存储敏感信息
- 使用 Gunicorn 等 WSGI 服务器代替 Flask 开发服务器
