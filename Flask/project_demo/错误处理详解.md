# Flask 错误处理详解

## 📚 什么是错误处理？

### 通俗理解

想象你去餐厅点菜：
- **正常情况**：点菜 → 上菜 → 吃饭 ✅
- **错误情况**：
  - 点了菜单上没有的菜 → 服务员说"这个我们没有"（404）
  - 用现金支付但只接受扫码 → 服务员说"不支持这种方式"（405）
  - 点了一道做不了的菜 → 厨师说"这个做不了"（400）
  - 厨房着火了 → 餐厅说"今天无法营业"（500）

**错误处理就是：当出现问题时，给用户一个友好的提示，而不是一堆看不懂的代码错误。**

---

## 🎯 为什么要用错误处理？

### 不使用错误处理（默认情况）

```
用户访问：http://127.0.0.1:5000/nonexistent

返回：
<!DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 3.2 Final//EN">
<title>404 Not Found</title>
<h1>Not Found</h1>
<p>The requested URL was not found on the server.</p>
```

❌ **问题**：
- 用户体验差（看到英文错误）
- 格式不统一（HTML 而不是 JSON）
- API 客户端无法解析

### 使用错误处理后

```json
{
  "code": 404,
  "message": "资源未找到",
  "error": "路径 /nonexistent 不存在"
}
```

✅ **优点**：
- 用户友好（中文提示）
- 格式统一（都是 JSON）
- 方便前端处理

## 📖 核心语法

### 基本用法

```python
from flask import Flask

app = Flask(__name__)

# 装饰器：@app.errorhandler(错误码)
@app.errorhandler(404)
def page_not_found(error):
    """处理 404 错误"""
    return '页面未找到', 404

@app.errorhandler(500)
def server_error(error):
    """处理 500 错误"""
    return '服务器错误', 500
```

### 在蓝图中使用

```python
from flask import Blueprint

error_bp = Blueprint('error', __name__)

# 注意：用 app_errorhandler 而不是 errorhandler
@error_bp.app_errorhandler(404)
def not_found(error):
    return '页面未找到', 404
```

**关键区别**：
- `@app.errorhandler()` - 在应用主文件中使用
- `@blueprint.app_errorhandler()` - 在蓝图中使用（全局生效）
- `@blueprint.errorhandler()` - 只在蓝图中生效

## 🔥 常见错误码详解

### 1. 400 Bad Request（错误请求）

**含义**：客户端发送的请求有问题

**触发场景**：
```python
# 场景 1：JSON 格式错误
requests.post('/api/user', data='不是 JSON')

# 场景 2：缺少必填参数
requests.post('/api/user', json={'username': 'test'})  # 缺少 password 等

# 场景 3：参数类型错误
requests.get('/api/user/abc')  # 应该是数字
```

**处理示例**：
```python
@error_bp.app_errorhandler(400)
def bad_request(error):
    return jsonify({
        'code': 400,
        'message': '错误的请求',
        'error': str(error.description)
    }), 400
```

---

### 2. 401 Unauthorized（未授权）

**含义**：需要登录才能访问

**触发场景**：
```python
# 访问需要登录的接口，但没有提供 token
requests.get('/api/user/profile')
```

**手动触发**：
```python
from flask import abort

@app.route('/profile')
def profile():
    if not user.is_authenticated:
        abort(401)  # 手动触发 401 错误
    return '用户资料'
```

---

### 3. 403 Forbidden（禁止访问）

**含义**：已登录，但没有权限

**触发场景**：
```python
# 普通用户访问管理员接口
requests.get('/api/admin/users')
```

**示例**：
```python
@app.route('/admin')
def admin():
    if not user.is_admin:
        abort(403)  # 权限不足
    return '管理后台'
```

---

### 4. 404 Not Found（未找到）

**含义**：请求的资源不存在

**触发场景**：
```python
# 场景 1：访问不存在的 URL
requests.get('/api/nonexistent')

# 场景 2：数据库记录不存在
User.query.get_or_404(99999)  # ID 不存在
```

**处理示例**：
```python
@error_bp.app_errorhandler(404)
def not_found(error):
    # 区分 API 和普通页面
    if request.path.startswith('/api/'):
        return jsonify({
            'code': 404,
            'message': '资源未找到'
        }), 404
    else:
        return '页面未找到', 404
```

---

### 5. 405 Method Not Allowed（方法不允许）

**含义**：请求方法不对

**触发场景**：
```python
# 接口只允许 POST，但用了 GET
@app.route('/user/create', methods=['POST'])
def create_user():
    pass

# 错误调用
requests.get('/user/create')  # ❌ 应该用 POST
```

---

### 6. 500 Internal Server Error（服务器错误）

**含义**：服务器内部出错

**触发场景**：
```python
# 场景 1：数据库连接失败
db.session.query(User).all()  # 数据库挂了

# 场景 2：代码逻辑错误
result = 1 / 0  # 除以零

# 场景 3：第三方服务异常
response = requests.get('http://api.example.com')  # 外部 API 挂了
```

**处理示例**：
```python
@error_bp.app_errorhandler(500)
def server_error(error):
    # 开发环境：返回详细错误（方便调试）
    if app.debug:
        return jsonify({
            'code': 500,
            'error': str(error)
        }), 500
    # 生产环境：返回简洁信息（安全考虑）
    else:
        return jsonify({
            'code': 500,
            'message': '服务器错误，请稍后重试'
        }), 500
```

## 🎨 项目中的错误处理

### 文件结构

```
app/
└── errors.py          # 错误处理蓝图（新增）
```

### 核心代码解析

#### 1. 创建错误处理蓝图

```python
from flask import Blueprint, jsonify, request

error_bp = Blueprint('error', __name__)
```

#### 2. 处理 404 错误

```python
@error_bp.app_errorhandler(404)
def not_found(error):
    """404 错误处理"""
    # 判断是 API 请求还是普通页面
    if request.path.startswith('/api/'):
        return jsonify({
            'code': 404,
            'message': '资源未找到',
            'error': f'路径 {request.path} 不存在'
        }), 404
    else:
        return jsonify({
            'code': 404,
            'message': '页面未找到'
        }), 404
```

**关键点**：
- `request.path`：获取当前请求的路径
- 根据路径判断返回不同格式
- 返回元组：`(响应内容, 状态码)`

#### 3. 处理 500 错误

```python
@error_bp.app_errorhandler(500)
def internal_server_error(error):
    """500 错误处理"""
    from flask import current_app
    
    # 开发环境 vs 生产环境
    if current_app.debug:
        return jsonify({
            'code': 500,
            'message': '服务器内部错误',
            'error': str(error),  # 详细错误信息
            'traceback': '...'     # 错误堆栈
        }), 500
    else:
        return jsonify({
            'code': 500,
            'message': '服务器内部错误，请稍后重试'
        }), 500
```

**关键点**：
- `current_app`：获取当前应用实例
- `current_app.debug`：判断是否调试模式
- 生产环境不暴露详细错误（安全）

#### 4. 捕获所有异常

```python
@error_bp.app_errorhandler(Exception)
def handle_exception(error):
    """捕获所有未处理的异常"""
    import traceback
    
    # 记录日志
    current_app.logger.error(f'错误: {str(error)}')
    current_app.logger.error(traceback.format_exc())
    
    return jsonify({
        'code': 500,
        'message': '服务器错误'
    }), 500
```

**关键点**：
- `Exception`：捕获所有异常类型
- 这是**最后一道防线**
- 记录日志方便排查问题

## 🎯 错误处理最佳实践

### 1. 统一的错误格式

所有错误都返回相同的 JSON 格式：
```json
{
  "code": 404,           // HTTP 状态码
  "message": "错误描述",  // 用户友好的提示
  "error": "详细信息"     // 开发环境可选
}
```

### 2. 区分环境

```python
# 开发环境：详细错误（方便调试）
if current_app.debug:
    return {'error': str(error), 'traceback': '...'}

# 生产环境：简洁信息（安全）
else:
    return {'message': '系统繁忙，请稍后重试'}
```

### 3. 记录日志

```python
@error_bp.app_errorhandler(Exception)
def handle_exception(error):
    # 记录错误日志
    current_app.logger.error(f'错误: {error}')
    current_app.logger.error(traceback.format_exc())
    
    return jsonify({'message': '服务器错误'}), 500
```

### 4. 主动抛出错误

```python
from flask import abort

@app.route('/user/<int:user_id>')
def get_user(user_id):
    user = User.query.get(user_id)
    if not user:
        abort(404)  # 主动触发 404 错误
    return jsonify(user.to_dict())
```

或者使用 `get_or_404()`：
```python
user = User.query.get_or_404(user_id)  # 不存在自动 404
```

## 📋 错误处理流程图

```
用户请求
    ↓
路由匹配？
    ├─ 否 → 404 错误处理
    └─ 是 ↓
         方法正确？
         ├─ 否 → 405 错误处理
         └─ 是 ↓
              参数正确？
              ├─ 否 → 400 错误处理
              └─ 是 ↓
                   有权限？
                   ├─ 未登录 → 401 错误处理
                   ├─ 无权限 → 403 错误处理
                   └─ 有权限 ↓
                        执行业务逻辑
                        ├─ 成功 → 返回数据
                        └─ 失败 → 500 错误处理
```

## 💡 常见问题

### Q1: `@app.errorhandler` 和 `@blueprint.app_errorhandler` 有什么区别？

**A**: 
- `@app.errorhandler()` - 只在当前应用生效
- `@blueprint.app_errorhandler()` - 全局生效（推荐在蓝图中使用）
- `@blueprint.errorhandler()` - 只在该蓝图中生效

### Q2: 如何手动触发错误？

**A**: 使用 `abort()` 函数：
```python
from flask import abort

abort(404)           # 触发 404
abort(500)           # 触发 500
abort(400, '错误信息')  # 带自定义信息
```

### Q3: 错误处理函数能返回 HTML 吗？

**A**: 可以！API 返回 JSON，网页返回 HTML：
```python
@app.errorhandler(404)
def not_found(error):
    if request.path.startswith('/api/'):
        return jsonify({'error': 'Not Found'}), 404
    else:
        return render_template('404.html'), 404
```

### Q4: 如何获取错误的详细信息？

**A**: 
```python
@error_bp.app_errorhandler(500)
def server_error(error):
    print(f'错误类型: {type(error)}')
    print(f'错误信息: {str(error)}')
    print(f'错误描述: {error.description}')  # 如果有
```

## 📚 总结

### 错误处理的核心要点

1. **统一格式**：所有错误返回相同的 JSON 结构
2. **用户友好**：中文提示，不说技术术语
3. **区分环境**：开发环境详细，生产环境简洁
4. **记录日志**：方便排查问题
5. **主动抛出**：在合适的地方使用 `abort()`
