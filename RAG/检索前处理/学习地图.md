# 检索前处理学习地图

## 章节目标

本章节只关注“进入检索前”的查询处理，不涉及向量库实现细节。你将掌握：

1. 查询翻译：查询重写、查询分解、查询澄清、查询扩展、HyDE。
2. 查询路由：逻辑路由、语义路由。
3. 如何把这些策略组织成可配置、可评估的前处理流水线。

## 总体流程

1. 用户原始查询
2. 查询意图判断（是否需要澄清）
3. 查询翻译（重写/分解/扩展/HyDE）
4. 查询路由（逻辑/语义）
5. 输出给对应检索器（向量检索、关键词检索、混合检索、多索引）

## 学习模块

### 01 查询翻译

覆盖知识点：

1. 查询重写（Query Rewrite）
2. 查询分解（Query Decomposition）
3. 查询澄清（Query Clarification）
4. 查询扩展（Query Expansion）
5. HyDE（Hypothetical Document Embeddings）

交付：

1. 教学文档：`01_查询翻译/README.md`
2. demo：`demos/demo_preprocess_pipeline.py`（对应函数）

### 02 查询路由

覆盖知识点：

1. 逻辑路由（规则/条件路由）
2. 语义路由（基于语义标签或向量相似度路由）

交付：

1. 教学文档：`02_查询路由/README.md`
2. demo：`demos/demo_preprocess_pipeline.py`（对应函数）

## 评估维度

1. 检索质量：Recall@K、MRR
2. 处理成本：额外 token、额外延迟
3. 稳定性：改写是否偏题、分解是否遗漏子问题、路由是否误判

## 推荐学习顺序

1. 先学查询翻译中的“重写 + 澄清”
2. 再学“分解 + 扩展 + HyDE”
3. 最后学“逻辑路由 + 语义路由”并串成流水线
